<task type="auto">
  <name>Task 2: Run WikiText-2 evaluation pipeline</name>
  <files>results/Phi-tiny-MoE-instruct/metrics.json, results/SUMMARY.md</files>
  <action>
Run the unified eval pipeline which measures FP16 baseline PPL, reconstructs the quantized model from factors, measures quantized PPL, and captures memory/size metrics:

```
python scripts/run_eval.py \
  --model microsoft/Phi-tiny-MoE-instruct \
  --quantized-dir output/Phi-tiny-MoE-instruct \
  --results-dir results
```

This command will:
1. Measure FP16 baseline WikiText-2 perplexity (dtype=auto)
2. Detect raw factors in quantized-dir (no config.json with weights = factors dir)
3. Auto-reconstruct to output/Phi-tiny-MoE-instruct_reconstructed/ via scripts/reconstruct_model.py
4. Measure quantized model WikiText-2 perplexity
5. Record peak VRAM, model sizes (FP16 vs quantized), compression ratio
6. Write results/Phi-tiny-MoE-instruct/metrics.json
7. Rebuild results/SUMMARY.md via build_summary.py

IMPORTANT: run_eval.py checks `is_factors_dir()` which requires quantized_factors.safetensors present AND no config.json in the same dir. However, output/Phi-tiny-MoE-instruct/ currently HAS config.json, so run_eval.py will treat it as a HF model dir (is_hf_model_dir returns True). This means it will try to load the quantized model directly from that directory. If config.json in output/Phi-tiny-MoE-instruct/ does NOT contain actual model weights (just metadata), the eval will fail on the quantized portion.

If the quantized eval fails because the output dir has config.json but no model weights: either (a) temporarily move config.json out so is_factors_dir detects correctly, or (b) run reconstruction manually first:
```
python -m scripts.reconstruct_model \
  --model microsoft/Phi-tiny-MoE-instruct \
  --quantized-dir output/Phi-tiny-MoE-instruct \
  --output output/Phi-tiny-MoE-instruct_reconstructed
```
Then point run_eval.py --quantized-dir at the reconstructed dir.

After eval completes, verify metrics.json contains all required fields.
