1. Change `FUSED_EXPERT_CLASSES` from a `frozenset` to a `dict` mapping class name to layout metadata:

```python
FUSED_EXPERT_CLASSES = {
    "Qwen2MoeExperts":   {"needs_split": False},
    "Qwen3MoeExperts":   {"needs_split": False},
    "Qwen3_5MoeExperts": {"needs_split": False},
    "OlmoeExperts":      {"needs_split": False},
    "PhimoeExperts":     {"needs_split": True, "split_dim": 0},
}
```

2. Update the docstring/comment above `FUSED_EXPERT_CLASSES` to explain the dict structure: each value is a dict with `needs_split` (bool) and optionally `split_dim` (int, the dimension of the 2D per-expert slice to split along for gate_up_proj).

3. In `get_weight_views`, update the fused expert branch. The `if type_name in FUSED_EXPERT_CLASSES:` check already works with a dict. Inside the loop over `("gate_up_proj", "down_proj")`:
   - Look up layout: `layout = FUSED_EXPERT_CLASSES[type_name]`
   - Check if this param needs splitting: `needs_split = layout.get("needs_split", False) and param_name == "gate_up_proj"`
   - If `needs_split`:
     - `half = param.shape[1] // 2` (dim 1 of 3D tensor is the out_dim, which is 2*intermediate)
     - For each expert_idx, yield TWO WeightViews:
       - `name=f"{mod_name}.gate_proj[{expert_idx}]"`, `weight=param.data[expert_idx, :half, :].float()`
       - `name=f"{mod_name}.up_proj[{expert_idx}]"`, `weight=param.data[expert_idx, half:, :].float()`
     - Write-back closures must write to the correct half:
       ```python
       def make_gate_write(p, idx, h):
           def _write(W_new): p.data[idx, :h, :] = W_new.to(p.dtype)
           return _write
       def make_up_write(p, idx, h):
           def _write(W_new): p.data[idx, h:, :] = W_new.to(p.dtype)
           return _write
       ```
   - If NOT `needs_split`: existing behavior unchanged (yield one WeightView per expert per param).

4. Update `get_hessian_key` to handle the new split view names. Currently it strips `[N]` suffix and looks for `.gate_up_proj` or `.down_proj` in the name. Split views will have `.gate_proj[N]` or `.up_proj[N]` instead. Add handling:
   - If `.gate_proj` in name: base = name[:name.index(".gate_proj")]
   - If `.up_proj` in name AND it's a fused split (contains `[`): base = name[:name.index(".up_proj")]
   - These both map to the same fused expert module Hessian key (same as gate_up_proj).

5. Update the `is_inside_experts` check in the nn.Linear branch — it currently does `type(ancestor).__name__ in FUSED_EXPERT_CLASSES` which works with dict (checks keys). No change needed here, just verify.

6. Update module docstring to mention PhimoeExperts.
