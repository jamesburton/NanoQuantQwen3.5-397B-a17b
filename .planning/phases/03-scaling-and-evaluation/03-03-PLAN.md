<task type="auto">
  <name>Task 3: Quantize and evaluate Qwen3.5-397B-A17B on cloud</name>
  <files>results/Qwen3.5-397B-A17B/metrics.json</files>
  <action>
On the cloud instance, run Stage 1 on Qwen3.5-397B-A17B. This is the primary target model.

**IMPORTANT — Known risk:** Qwen3.5-397B uses a hybrid Gated DeltaNet + MoE architecture. The `_run_block` in reconstruct.py may fail on DeltaNet layers. Run a 5-block dry run first to exercise both DeltaNet and Attention block types:

```bash
python scripts/run_stage1.py \
  --model Qwen/Qwen3.5-397B-A17B \
  --output output/Qwen3.5-397B-A17B \
  --bits-per-weight 0.80 \
  --admm-iters 50 \
  --n-calibration 128 \
  --seq-len 2048 \
  --max-blocks 5 \
  --checkpoint-dir checkpoints/Qwen3.5-397B-A17B
```

If dry run fails on DeltaNet blocks (AttributeError or shape mismatch in reconstruct.py), document the error and add a try/except fallback in `_call_block` that calls `block(hidden_states)` directly for non-standard block types. Then retry.

If dry run succeeds, run full quantization (remove `--max-blocks`).

Then run evaluation:

```bash
python scripts/run_eval.py \
  --model Qwen/Qwen3.5-397B-A17B \
  --quantized-dir output/Qwen3.5-397B-A17B \
  --results-dir results \
  --stride 512 \
  --max-length 2048
```

Note: Use `--dtype auto` for the FP16 baseline eval since Qwen3.5-397B is natively bfloat16.

Copy `results/Qwen3.5-397B-A17B/metrics.json` back to local machine.

After both cloud models complete, rebuild SUMMARY.md locally with all 4 models:

```bash
python scripts/build_summary.py --results-dir results
```
