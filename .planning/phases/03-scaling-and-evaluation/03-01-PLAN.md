  <name>Task 1: Rewrite eval_ppl.py with canonical sliding-window PPL</name>
  <files>scripts/eval_ppl.py</files>
  <action>
Rewrite `evaluate_perplexity()` in `scripts/eval_ppl.py` to use the HuggingFace canonical sliding-window method:

1. Replace the current manual logits+CrossEntropyLoss approach with `model(input_ids, labels=target_ids)` using `-100` masking for context tokens.

2. Use these defaults: `stride=512`, `max_length=2048` (configurable via CLI args).

3. Key implementation (from HF docs):
   - For each window: `target_ids = input_ids_window.clone()`, then `target_ids[:, :-trg_len] = -100`
   - Use `outputs.loss` (internally shifted by transformers) weighted by number of non-masked tokens
   - Accumulate `nll_sum` and `n_tokens`, compute `ppl = exp(nll_sum / n_tokens)`

4. Add `--stride` CLI arg (default 512). Keep `--max-length` but default to 2048 instead of 32768.

5. Support `torch_dtype="auto"` for baseline eval (respects model's native dtype, e.g. bfloat16 for Qwen3.5-397B). Add `--dtype` arg: "auto" (default for baseline) or "float16" (for quantized).

6. Return a dict from `evaluate_perplexity()` instead of just float: `{"ppl": float, "n_tokens": int, "settings": {"stride": int, "max_length": int}}`.

7. Keep the CLI interface working (`--model`, `--baseline`, `--device`).

Do NOT truncate the full WikiText-2 test set — encode without truncation, then use the sliding window over the full sequence.

<task type="auto">
  <name>Task 2: Create run_eval.py unified evaluation runner with JSON metrics</name>
  <files>scripts/run_eval.py, scripts/build_summary.py</files>
  <action>
Create `scripts/run_eval.py` — a unified evaluation script that runs FP16 baseline PPL, quantized PPL, captures memory/timing metrics, writes JSON, and rebuilds SUMMARY.md.

**run_eval.py:**

CLI args:
- `--model` (required): HF model name (e.g. `allenai/OLMoE-1B-7B-0924`)
- `--quantized-dir` (required): path to quantized output dir from run_stage1.py
- `--results-dir` (default `results`): where to write results
- `--device` (default `cuda`): compute device
- `--stride` (default 512), `--max-length` (default 2048): PPL eval settings

Flow:
1. Create `results/{model_short_name}/` directory (model_short_name = last part of HF name, e.g. `OLMoE-1B-7B-0924`)
2. Call `torch.cuda.reset_peak_memory_stats()` if CUDA available
3. Run FP16 baseline PPL via `evaluate_perplexity(model, dtype="auto")`
4. Run quantized PPL via `evaluate_perplexity(quantized_dir, dtype="float16")`
5. Capture memory metrics:
   - `peak_vram_gb`: `torch.cuda.max_memory_allocated() / 1024**3` (0 if no CUDA)
   - `peak_ram_gb`: `psutil.virtual_memory().used / 1024**3`
   - `gpu_name`: `torch.cuda.get_device_name(0)` if available
6. Capture model size metrics:
   - `fp16_gb`: sum of .safetensors/.bin files in HF cache or model dir
   - `quantized_gb`: sum of .safetensors/.bin/.pt files in quantized_dir
   - `compression_ratio`: fp16 / quantized
7. Read quantization config from `quantized_dir/quantized_manifest.json` if exists
8. Write metrics dict to `results/{model_short_name}/metrics.json` using the JSON schema from RESEARCH.md (model, model_type, date, hardware, model_size, timing, perplexity, quantization_config, stage1_complete, loadable_checkpoint)
9. Call `build_summary.py` logic to rebuild SUMMARY.md

Import `evaluate_perplexity` from `scripts.eval_ppl` (or use subprocess if import path issues — add `sys.path` adjustment if needed).

**build_summary.py:**

Standalone script that reads all `results/*/metrics.json` files and generates `results/SUMMARY.md` with comparison table.

Table columns: Model | FP16 PPL | Quant PPL | Degradation | FP16 Size | Quant Size | Ratio | Peak VRAM | Quant Time

Make `rebuild_summary()` importable so run_eval.py can call it directly.

Both scripts should print clear error messages if run on insufficient hardware.
